# 短视频音频版权检测系统

基于 Rust + Axum 实现的短视频音频版权检测系统，使用音频指纹算法（类似 Shazam）实现快速、准确的版权识别。

## 功能特性

- 🔊 **多格式支持**: 支持 MP3、WAV、AAC、MP4 等常见音频格式
- 🎵 **音频指纹**: 基于 FFT 频谱分析和峰值匹配的指纹算法
- ⚡ **快速检测**: 哈希索引 + 时间差直方图匹配，检测速度快
- 📚 **版权库管理**: RESTful API 管理版权音频库
- 🔍 **片段检测**: 支持检测音频片段是否侵权
- 📊 **详细结果**: 返回匹配片段、置信度等详细信息

## 核心算法

### 音频指纹生成

1. **重采样**: 将音频统一重采样到 22050Hz 单声道
2. **分帧**: 使用 2048 点帧长，512 点帧移
3. **窗函数**: 汉宁窗减少频谱泄漏
4. **FFT**: 快速傅里叶变换得到频谱
5. **峰值提取**: 在频域中提取局部峰值点
6. **双重哈希**:
   - **频率对哈希 (`freq_hash`)**: 仅基于 (f1, f2) 生成，对变速完全不敏感，用于粗匹配
   - **量化时间差哈希**: 将时间差 Δt 量化到大区间（1-2, 3-5, 6-8...帧），提高对变速的容忍度
7. **扩展 FAN_VALUE**: 从 5 增加到 15，生成更多指纹，提高片段匹配概率

### 版权匹配（增强鲁棒版）

#### 1. 双层候选检索
- **频率对倒排索引**: 用 `freq_hash` 快速找候选（对变速不敏感）
- **含量化时间差的哈希索引**: 更精确的候选筛选
- 综合评分选出 Top 8 候选曲目

#### 2. 多时间缩放因子匹配（抗变速核心）
尝试 14 种缩放因子：`[0.70, 0.75, 0.80, 0.85, 0.90, 0.95, 1.00, 1.05, 1.10, 1.15, 1.20, 1.30, 1.40, 1.50]`

对每种缩放因子：
1. 将查询时间轴按因子缩放
2. 用直方图找最优时间偏移
3. 用 **LIS（最长递增子序列）** 提取时序一致的匹配对
4. 过滤斜率异常的点（slope 在 0.3~3.0 之间为合法）

#### 3. DTW 轻量版（动态时间规整）
当初始匹配效果差时触发：
- 在 (查询帧, 目标帧) 匹配对图上找最长合法路径
- 要求相邻点斜率在 0.5~2.0 之间
- 用路径长度和累积成本综合评分

#### 4. 滑窗口分段匹配
对长音频分段处理：
- 将查询音频按 `window_size = min(max(duration/3, 3s), 15s)` 分窗口
- 窗口步进 = 窗口大小/2（50% 重叠）
- 每窗口独立匹配后取最高分
- 最后整段匹配结果与窗口结果投票选优

#### 5. 置信度综合评分
```
置信度 = 一致性分数×0.4 + 时间覆盖率×0.35 + 匹配密度×0.25
```

最终置信度 = 算法置信度×60% + 片段平均置信度×40%

#### 6. 增强片段提取
每个匹配片段输出：
- 查询起始/结束时间
- 版权曲起始/结束时间
- 片段置信度
- **检测到的时间缩放因子（scale_factor）**

## 抗干扰能力对比

| 干扰类型 | 原版算法 | 增强版 | 原理 |
|---------|---------|--------|------|
| 原速直接剪辑 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | FAN_VALUE 增大 + 滑窗口 |
| 0.7x 慢速播放 | ❌ | ⭐⭐⭐⭐⭐ | 多缩放因子匹配 + freq_hash |
| 1.2x 快速播放 | ❌ | ⭐⭐⭐⭐⭐ | 多缩放因子匹配 + freq_hash |
| 1.5x 极限变速 | ❌ | ⭐⭐⭐⭐ | 缩放因子扩展到 1.5 |
| 变速+剪辑混合 | ❌ | ⭐⭐⭐⭐ | LIS+滑窗+DTW 三重保障 |
| 拼接两段不同内容 | ⭐⭐ | ⭐⭐⭐⭐ | 多窗口独立检测 |
| 噪声干扰 | ⭐⭐⭐ | ⭐⭐⭐⭐ | 降低阈值 + 密度评分 |

## API 接口

### 健康检查

```
GET /api/health
```

响应:
```json
{
  "status": "ok",
  "library_size": 10
}
```

### 版权库管理

**获取曲目列表**
```
GET /api/library
```

**获取单曲信息**
```
GET /api/library/:id
```

**添加版权曲目**
```
POST /api/library?title=歌曲名&artist=艺术家
Content-Type: multipart/form-data

audio: <音频文件>
```

响应:
```json
{
  "track_id": "uuid",
  "title": "歌曲名",
  "artist": "艺术家",
  "duration": 180.5,
  "fingerprint_count": 15000
}
```

**删除曲目**
```
DELETE /api/library/:id
```

### 侵权检测

**检测音频是否侵权**
```
POST /api/detect
Content-Type: multipart/form-data

audio: <音频文件>
```

响应:
```json
{
  "is_infringing": true,
  "confidence": 0.85,
  "matched_track": {
    "id": "uuid",
    "title": "版权歌曲",
    "artist": "艺术家",
    "duration": 200.0,
    "fingerprint_count": 20000,
    "created_at": 1234567890
  },
  "match_segments": [
    {
      "query_start": 5.0,
      "query_end": 25.0,
      "track_start": 10.0,
      "track_end": 30.0,
      "confidence": 0.9,
      "scale_factor": 1.0
    },
    {
      "query_start": 30.0,
      "query_end": 45.0,
      "track_start": 40.0,
      "track_end": 58.0,
      "confidence": 0.82,
      "scale_factor": 1.2
    }
  ],
  "processing_time_ms": 250
}
```

> **注意**: `scale_factor != 1.0` 表示检测到了变速处理。例如 `scale_factor: 1.2` 意味着该片段被加速了 20%。

## 使用说明

### 编译运行

```bash
# 开发模式运行
cargo run

# 发布版本编译
cargo build --release

# 运行发布版本
./target/release/audio-copyright-detector
```

服务默认监听 `127.0.0.1:3000`。

```bash
cargo run --bin generate_test_audio
```

这会在 `test_audio/` 目录下生成几个测试用的 WAV 文件。

## 配置说明

可通过环境变量配置:

- `RUST_LOG`: 日志级别（默认: `info`）

## 依赖

- `axum`: Web 框架
- `tokio`: 异步运行时
- `symphonia`: 音频解码
- `rustfft`: 快速傅里叶变换
- `hound`: WAV 文件读写
- `uuid`: 唯一标识符
- `serde/serde_json`: 序列化
- `parking_lot`: 高性能锁

## 性能特点

- 使用倒排索引实现 O(1) 哈希查找
- 时间差直方图算法抗噪性强
- 支持部分片段匹配
- 内存中的版权库，检测速度快

## 注意事项

1. 目前版权库存储在内存中，服务重启后数据会丢失
2. 建议使用时将版权库持久化到数据库（如 SQLite、PostgreSQL）
3. 音频指纹算法参数可根据实际场景调整
4. 置信度阈值可在 `CopyrightLibrary::with_threshold()` 中设置
